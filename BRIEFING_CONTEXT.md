# Daily Brief 컨텍스트

## 목적
매일 아침 받는 종합 브리핑. 권위 있는 소스에서 가져온 뉴스를 Claude가 분석해서 슬랙으로 전달.

## 받는 사람
한국어 네이티브, 대전 거주, 한국 시간 기준 아침 7시에 읽음.
경제·정치·기술/AI 전반에 대한 객관적 정보를 빠르게 파악하고 싶어함.

## 관심 영역

### 메인 (균형 있게 다룰 것)
- **경제 / 시장**: 주요 지수, 환율, 금리, 글로벌·한국 시장 동향
- **경제 분석 / 예측**: 권위 있는 은행·애널리스트·증권사 리서치, 시장 변화 예측
- **정치**: 글로벌 주요 사건, 한국 주요 정치/정책 이슈
- **유명인 시그널**: 경제·기술 분야 권위자들의 주목할 만한 발언 (현재는 RSS 기반 뉴스에서 인용된 것 위주)

### 별도 섹션 — 깊게 다룰 것
- **기술 / AI**: 새 모델 발표, AI 도구, 빅테크 동향, AI 회사 1차 소스(Anthropic, OpenAI 등)
  - 다른 카테고리보다 더 자세하게
  - 1차 소스(회사 공식 발표) 우선
  - 기술적 의미 + 산업 영향 모두

### 무시
- 연예/스포츠 가십
- 자극적 클릭베이트 제목
- 광고성 PR

## 권위 / 신뢰도 우선순위

데이터 소스가 여러 개 같은 사건을 다룰 때 우선순위:

**티어 1 (가장 신뢰)**
- 1차 소스: 회사·기관 공식 발표 (Anthropic News, OpenAI News, 정부 발표)
- 권위 통신사: Reuters, FT, BBC

**티어 2 (신뢰)**
- 한국 경제 매체: 한국경제, 매일경제, 연합뉴스
- 글로벌 테크: TechCrunch
- 한국 테크: 전자신문

**티어 3 (참고)**
- 기타 매체

같은 사건이 여러 곳에서 나오면 티어 높은 것을 메인으로, 다른 건 보완 자료로.

## 분석 스타일

### 형식
- 각 뉴스 항목: 2~3줄 이내, 한국어
- 제목은 굵게 + 원문 링크
- "왜 중요한가" 한국어로 명료하게
- 분석/예측이 있는 뉴스는 그 핵심을 강조

### 톤
- 객관적, 분석적
- 감정적 표현 자제
- 추측 시 "~로 해석됨", "~가능성 제기" 같은 명확한 표시
- 사실과 의견 구분

### 길이
- 압축 우선. 아침에 받는 메시지라 길면 부담
- 핵심만 추리고 나머지는 과감히 제외
- 슬랙 4000자 제한 준수, 그 안에서 깔끔하게

## 능동적 분석

- 수집된 RSS 기사 외에 더 깊은 맥락 필요하면 WebSearch 사용 (최대 3회)
- 같은 주제 여러 매체가 다루면 종합해서 한 번에 정리
- 분석/예측 리포트가 있으면 그 핵심 강조
- 상관관계 보이면 짚어주기 ("X 발표 → Y 시장 반응")
